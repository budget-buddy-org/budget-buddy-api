Scaffold a new CRUDL domain feature end-to-end. Accepts the domain name as the argument (e.g. `/new-feature budget`).

**Usage:** `/new-feature <domain-name>`

**Steps:**

1. Derive names from the argument:
   - Domain: `<domain-name>` (e.g. `budget`)
   - Package: `com.budget.buddy.budget_buddy_api.<domain-name>`
   - Entity class: `<Domain>Entity`
   - DTO class: `<Domain>` (as generated by OpenAPI)
   - Controller interface: `<Domain>sApi` (as generated by OpenAPI)

2. **Spec first** ‚Äî open `src/main/resources/openapi.yaml` and confirm that the required paths and schema for this domain already exist. If they are missing, stop and tell the user to add the spec entries before proceeding.

3. **Generate** ‚Äî run `./gradlew openApiGenerate` to regenerate models and interfaces from the updated spec.

4. **Liquibase migration** ‚Äî determine the next migration number by listing files in `src/main/resources/db/changelog/` and incrementing the highest number. Create the migration file `src/main/resources/db/changelog/<next-number>_create_<domain-name>_table.sql`. Include at minimum: `id` (UUID primary key), `owner_id` (UUID not null), `created_at`, `updated_at`, plus domain-specific columns. Also register the new file in `src/main/resources/db/changelog/db.changelog-master.yaml`.

5. **Entity** ‚Äî create `src/main/java/com/budget/buddy/budget_buddy_api/<domain-name>/<Domain>Entity.java` implementing `OwnableEntity`. Use Lombok `@Data`, `@Table`. Map every column with `@Column` where needed.

6. **Repository** ‚Äî create `<Domain>Repository` extending `BaseEntityRepository<<Domain>Entity>`.

7. **Mapper** ‚Äî create `<Domain>Mapper` extending `BaseEntityMapper<<Domain>Entity, <Domain>>`. Annotate with `@Mapper(componentModel = "spring")`.

8. **Service** ‚Äî create `<Domain>Service` extending `OwnableEntityService<<Domain>Entity, <Domain>, <Domain>Repository>`. Wire in validators as a `Set<BaseEntityValidator<<Domain>Entity>>`.

9. **Controller** ‚Äî create `<Domain>Controller` implementing the generated `<Domain>sApi` and extending `BaseEntityController<<Domain>Entity, <Domain>>`. Override `createdURI()` to return the correct resource path. Delegate all methods to the `*Internal()` helpers.

10. **Integration tests** ‚Äî create `src/integrationTest/java/com/budget/buddy/budget_buddy_api/<domain-name>/<Domain>IT.java` extending `BaseMvcIntegrationTest`. Cover: create (201), read (200), update (200), delete (204), list (200), and auth guard (401 when no token).

11. **Compile check** ‚Äî run `./gradlew compileJava compileTestJava compileIntegrationTestJava` and fix any errors before finishing.

12. Report what was created and remind the user to run `./gradlew integrationTest` to validate end-to-end.

**Conventions to follow:**
- `var` for all local variable declarations
- No error handling for scenarios the framework already covers
- No extra abstractions beyond what is described above
- Throw `EntityNotFoundException` from service for missing entities ‚Äî do not catch in controller
